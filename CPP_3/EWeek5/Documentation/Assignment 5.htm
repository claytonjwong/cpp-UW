    Assignment 5

    Submit Assignment
    • Due Sep 16 by 11:55pm
    • Points 25
    • Submitting a file upload

    In this assignment, you will integrate the bitmap CODEC support from Assignment 4 into the vector graphics system and implement the new classes needed to render vector graphic scenes to a bitmap file.

    The first thing you might notice if you implemented this system without making any modifications to your old code is that the windows bitmap format actually stores scan lines in bottom-top-top order. This wasn't clear in Assignment 4, because there you were simply applying a modification to bitmaps; so whether it was upside down or right side up, your modifications would work the same. For this assignment, you will need to flip the order of scan lines when writing to the Windows Bitmap File format. You can use std::stack to facilitate this flipping.

    Since our system is only interested in rendering to and saving bitmaps, we will only need the CodecLibrary and WindowsBitmapEncoder (and their dependencies) in this assignment. Make sure when integrating the CODEC work that you namespace appropriately. Determine if the CODEC classes are a sub-namespace of our framework or a separate one, or perhaps just belong directly in the framework's namespace. You will note that the IBitmapIterator interface is the only thing the framework actually depends on.

    You will need to create a BasicCanvas class that implements the ICanvas interface, as well as a BitmapFileProjector that implements the IProjector interface. The BasicCanvas can be implemented well as an associative point-to-color mapping. This will be efficient in cases where there is a great deal of unused whitespace, since the BasicCanvas can just return the background color (which should be parameterized to it on construction) in these cases.

    In order to use Point class instances in a map, you will need to define a less-than operator for it or write a comparator. When writing whichever you choose, remember that comparators (and the less-than operator) must behave in a way that allows the operands to be switched to determine other forms of inequality (such as greater than or equal to).

    To test your system, you should create an XML file that exercises all major features of the system. Make sure you test both normal and exceptional situations, and that all exceptions are reported cleanly and clearly to the client.

    With your code submittal, make sure to include class diagrams of the current system.
