    Answers to Practice Questions for Lesson 5

    Answers to Practice Exercises for Lesson 5

    1. C++ structures can use access specifiers and have both data and function members. When declaring an instance of a C++ structure, we do not have to use the keyword struct.
    2. Other than the name, the only difference between a C++ structure and a class is the default access specification. For a structure, it's public; for a class, it's private.
    3. Access specifiers identify the level of access to data and function members in a class or structure. Public members are visible to the world; protected members are visible only to members of the class, members of derived classes, or friends; and private members are visible only to members of the class or friends. The access specifiers allow us to implement information hiding.
    4. The public interface reflects how a class appears to the outside world. Contained therein are the function members other classes can use to read or write (hidden) internal data.
    5. You initialize a class by invoking one of several possible constructors. The default constructor takes no arguments and will be created by the compiler if no constructors are specified.

      Arguments passed into the constructor are used to initialize the data members via the init list. A constructor function can also have default parameters that can be used if it is invoked with fewer than the specified number of parameters.
    6. No. The destructor is used to deinitialize memory, and the delete operator is used to delete memory allocated from the heap. The destructor is automatically called by the delete operator before freeing storage.
    7. The C++ language supports three different name scopes: file, local, and class.
    8. Yes, we overload member functions in the same way as non-member functions. Such overloaded functions define what we call compile time polymorphism.
